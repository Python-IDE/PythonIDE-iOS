widget / 小组件

时间线和动画

Timeline entry、numericText、transition 和 flip。

Widget 动画由 WidgetKit 在数据更新时触发。它不是普通 App 的 60fps 动画，也不会持续循环。

#数字变化

预期效果：点击按钮后数字用 numericText 过渡；small 尺寸自动使用较小字号。

#时间线 entry

预期效果：时间线 entry 更新时显示翻页数字块；不同 family 使用不同翻页尺寸。

#能用的动画

  • .content_transition("numericText")：数字变化。
  • .transition("push", edge="bottom")：节点插入或替换。
  • .animation("smooth", value_by="progress")：对指定数据变化使用系统动画。
  • .id("stable-id")：给变化节点稳定身份。
  • w.flip(...)：翻页数字块，适合时间线或状态变化测试。

桌面小组件什么时候刷新由 WidgetKit 决定。排查时如果桌面仍显示旧内容，重新运行并发布；必要时删除桌面小组件后重新添加。

#失败路径

  • 动画没有连续播放：Widget 只在 timeline entry 或 AppIntent 状态变化时更新，不适合 60fps 循环动画。
  • 数字没动画：给数值节点加 .content_transition("numericText") 和 .monospaced_digit()，变化节点需要稳定身份时再加 .id(...)。
  • 时间线不刷新：确认 entry 的 date 是 ISO 字符串，并且 update/after/interval 符合你希望的刷新策略。
  • 预览和桌面不同步：重新运行并发布脚本；桌面仍旧时删除小组件后重新添加。

#API 参考

时间线和动画完整签名见 API 参考：重点查 w.timeline、widget.entry、w.flip、.content_transition()、.transition()、.animation() 和 .id()。
