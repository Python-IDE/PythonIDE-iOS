haptics

触觉反馈和 Core Haptics 模式。

haptics 用来在用户明确操作后触发 iOS 触觉反馈。它适合增强“选中、保存成功、警告、错误、轻重点击”等瞬时反馈，不应该替代文字、颜色、HUD 或状态变化。

#预期效果

示例会展示按钮触发 impact、selection、notification 和设备支持判断。

#什么时候用

  • 点击按钮、切换选项、拖动到关键位置时给出轻触感。
  • 保存成功、校验失败、危险操作被拦截时给出明确反馈。
  • 需要 Core Haptics 自定义短模式时，用 play(events) 播放 transient 或 continuous 事件。

#核心模型

impact、selection、notification 这一组是普通 UIKit 触觉生成器，适合绝大多数按钮和状态反馈。play(events) 使用 Core Haptics，只在设备支持时才应该调用，因此要先检查 is_supported()。

触觉只是辅助反馈。按钮点击后仍然要更新 AppUI State、显示提示文字或修改界面状态，否则用户在没有触觉的设备上会看不到结果。

#AppUI 中怎么接

把触觉放在按钮回调里，同时更新界面状态。不要在 body() 里直接触发触觉，否则每次刷新界面都可能重复震动。

#失败路径

情况应该怎么处理
设备不支持 Core Haptics跳过 play(events)，保留可见 UI 反馈。
用户关闭系统触觉不要把触觉作为唯一反馈。
高频循环触发改成关键状态变化时触发一次。
退出页面或停止模式调用 stop() 结束 Core Haptics 引擎。

#常见错误

错误写法后果修正
在 body() 里调用 haptics.success()界面重建时重复触发放进按钮或明确事件回调。
只触发触觉但不更新 UI静音或不支持设备上没有反馈同时更新 State 或显示 HUD。
不检查 is_supported() 就调用 play部分设备无效果或返回失败先判断支持情况。

#API 参考

类型API签名说明
functionimpactimpact(style: str = "medium", intensity: float = 1.0) -> None触发点击冲击反馈；style 可用 light、medium、heavy、soft、rigid。
functionnotificationnotification(type: str = "success") -> None触发通知类反馈；type 可用 success、warning、error。
functionselectionselection() -> None触发轻微选择变化反馈。
functionsuccesssuccess() -> Nonenotification("success") 的快捷方式。
functionwarningwarning() -> Nonenotification("warning") 的快捷方式。
functionerrorerror() -> Nonenotification("error") 的快捷方式。
functionlightlight() -> Noneimpact("light") 的快捷方式。
functionmediummedium() -> Noneimpact("medium") 的快捷方式。
functionheavyheavy() -> Noneimpact("heavy") 的快捷方式。
functionis_supportedis_supported() -> bool当前设备是否支持 Core Haptics 自定义模式。
functionplayplay(events: list[dict[str, Any]]) -> bool播放 Core Haptics 事件列表；事件包含 type、time、duration、intensity、sharpness。
functionstopstop() -> None停止 Core Haptics 引擎。
