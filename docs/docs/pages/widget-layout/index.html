布局与尺寸

row、column、grid、table、canvas 和三尺寸适配。

小组件布局以 family 为边界。先确认目标尺寸，再决定信息密度；不要把 large 布局直接缩进 small。

#基础布局

预期效果：medium 尺寸显示一行两个数值和一条进度条；每个数值单元保持单行并可缩放。

常用容器：

API用途
row()横向排列
column()纵向排列
layer()叠放
grid()等列网格
table()精确表格线
canvas()点坐标绘制

#三尺寸适配

预期效果：small 尺寸只显示标题和主值；medium/large 尺寸显示两个数值和进度条。

#表格线

表格、Bingo、日历这类作品优先用 table()，不要用一堆粗矩形硬拼线。

预期效果：九宫格使用细线表格，按钮文字在每个格子内居中并保持单行。

表格线建议：

  • line_width="hairline"：像素级细线。
  • line_cap="butt"：端点不变圆。
  • line_join="miter"：交点保持直角。

#失败路径

  • small 里文字被裁切：只保留标题、主值和一个视觉元素，并给长文本加 .line_limit(1).min_scale(0.6)。
  • medium 正常但 large 太空：用 w.when("large") 或 widget.family_value(...) 增加额外行、图表或表格内容。
  • 表格线发粗或交点圆：优先调 line_width="hairline"、line_cap="butt"、line_join="miter"，不要用矩形拼线。
  • canvas 内容跑出边界：读取 w.context.content_width 和 w.context.content_height，按点位计算后再 .place(...)。
