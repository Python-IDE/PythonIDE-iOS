widget / 小组件

排错

空白、溢出、桌面缓存、点击和动画问题。

#空白

空白通常来自脚本没有产出可显示的小组件内容。先确认脚本最后调用了 w.render()，并且没有在 Widget 脚本里混用 AppUI 或网页框架。

预期效果：这是排错用的最小健康样例；如果它能预览，问题通常在后续参数、资源、布局或时间线代码中。

#快速定位

现象先查继续看
空白w.render()、只用 widget 模块widget
挤压或裁切family 分支、.line_limit(1).min_scale(...)布局与尺寸
参数不见widget.param.* 声明位置和默认值类型参数面板
点击没反应widget.state action、按钮/链接层级状态和交互
动画不动timeline 或状态变化是否真的发生时间线和动画
外观不对深色、透明、染色、资源路径资源与外观

#小尺寸太挤

为 small 写单独布局，只保留标题、主值和一个视觉元素。不要把 large 的布局整体缩小。

#表格线太粗

使用 line_width="hairline"，不要用大矩形叠出来假线。

#字体溢出

默认给用户可改文本、按钮标题和主值加 .line_limit(1).min_scale(0.6)。仍然拥挤时，按 family 缩短文案或隐藏次要节点。

#预览和桌面不一致

先重新运行并发布脚本。WidgetKit 会缓存桌面显示结果；如果桌面仍旧，删除桌面小组件后重新添加。

#点击没反应

按钮必须使用 widget.state 生成的 action，例如 count.increment()、done.toggle() 或 items.toggle_item(i)。

#链接打开 App

桌面 Widget 的链接行为受系统和 URL scheme 限制。HTTP 链接用 w.link()；App 内页面用深链。

#动画不连续播放

WidgetKit 只支持 timeline 或 AppIntent 状态变化时的数据更新动画。持续 60fps 动画不属于普通桌面 Widget 能力。

#失败路径

  • 先做最小复现：保留 Widget()、两三个节点和 w.render()，确认预览能显示后再加参数、状态、图片和时间线。
  • 再查目标 family：small、medium、large、rectangular、circular、inline 的内容预算不同，不要用同一个密集布局硬套。
  • 最后查系统缓存：桌面小组件可能缓存旧时间线，重新运行并发布；仍旧时删除桌面小组件后重新添加。

#API 参考

排错时查 API 参考 的完整签名，优先确认 Widget()、w.render、w.validate、w.timeline、widget.param、widget.state 和布局修饰符是否写对。
