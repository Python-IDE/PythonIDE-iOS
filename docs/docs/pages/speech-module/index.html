speech

文字转语音和语音列表。

文字转语音和语音列表。

#预期效果

示例会展示文字转语音、语音列表和停止朗读的控制流程。

#适用场景

文字转语音、暂停恢复和可用语音列表。适合朗读结果、无障碍提示、学习卡片和短句反馈；长文本朗读必须提供停止入口。

#核心模型

say(text, language=None, rate=0.5, pitch=1.0, volume=1.0, wait=False) 会朗读文本。language 使用 BCP-47 标签，例如 en-US、zh-CN。AppUI 回调中通常保持 wait=False，否则界面会被朗读阻塞。

pause()、resume()、stop() 控制当前朗读队列。离开页面、开始新的长文本朗读或用户点击停止时，应调用 stop()。

#API 参考

类型API签名说明
functionsaysay(text, language=None, rate=0.5, pitch=1.0, volume=1.0, wait=False) -> Any朗读文本。
functionstopstop() -> Any立即停止朗读。
functionpausepause() -> Any暂停朗读。
functionresumeresume() -> Any恢复暂停的朗读。
functionis_speakingis_speaking() -> bool当前是否正在朗读。
functionavailable_voicesavailable_voices() -> Any返回可用语音标识列表。

#使用规则

  • 长文本朗读提供停止入口。
  • 需要同步等待结果时传 wait=True。
  • AppUI 回调里通常不要 wait=True，否则会阻塞交互。
  • 离开页面或开始新的朗读前可先调用 stop()。
  • available_voices() 可用于设置页，但不要每次重建界面都调用。

#失败路径

情况应该怎么处理
文本为空不调用 say，提示输入内容。
长文本朗读提供停止按钮，避免用户无法打断。
AppUI 回调中需要朗读使用 wait=False。
用户离开页面调用 stop()。

#常见错误

错误写法后果修正
AppUI 按钮里 wait=True界面阻塞使用 wait=False。
用 speech 播放音效能力边界错误使用 sound。
开始新朗读前不停止旧内容多段语音排队或重叠先 stop() 再 say()。
