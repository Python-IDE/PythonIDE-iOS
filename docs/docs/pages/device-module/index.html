iOS 原生模块 / 设备与传感器

device

设备、屏幕、电池和系统状态。

device 读取当前 iPhone/iPad 的设备、系统、屏幕、电池、内存、温度、区域和语言信息。它主要用于适配界面、诊断环境、展示状态，不应该作为安全判断的唯一依据。

#API 参考

API返回用途
name()str设备名称。
model()str设备型号标识或可读型号。
system_name() / system_version()str系统名称和版本。
identifier_for_vendor()str当前开发者维度的设备标识；可能因重装/供应商变化而变化。
battery_level()float电量 0.0 到 1.0；不可用时可能为 -1.0。
battery_state()strunknown、unplugged、charging、full。
orientation()str当前方向。
screen_width() / screen_height()float屏幕点数宽高。
screen_size()tuple[float, float](width, height)。
screen_scale()float屏幕 scale。
screen_brightness() / set_screen_brightness(value)float / None读取或设置亮度，值为 0.0 到 1.0。
total_memory()int物理内存字节数。
processor_count()intCPU 核心数。
system_uptime()float开机以来秒数。
thermal_state()strnominal、fair、serious、critical。
is_low_power_mode()bool是否低电量模式。
locale() / timezone()str当前区域和时区。
preferred_languages()list[str]用户首选语言列表。

#失败路径

情况应该怎么处理
权限被拒绝显示当前状态和授权说明，不要循环弹出系统授权。
用户取消系统面板处理 None、False、空列表或错误字典，保持页面可继续操作。
设备或系统版本不支持显示不可用状态，并提供替代流程或禁用对应按钮。
返回数据为空把空状态写回界面，不把空结果当作异常崩溃。

#使用规则

  • 改亮度前要有明确用户动作，避免脚本自动改变设备体验。
  • 电池、温度、低电量模式可以用于降低刷新频率或延迟下载。
  • 屏幕尺寸用于布局适配，不要写死某一台设备。
  • identifier_for_vendor() 不是登录、支付或安全授权凭据。
