iOS 原生模块 / 媒体与视觉

photos

相册选择、拍照、保存和资源查询。

photos 用来选图、拍照、保存图片和视频、读取照片资源、管理相册，并把图片转换为 Base64。它默认走系统照片/相机界面，用户可以取消，因此每个流程都要处理 None、False 或空列表。

#预期效果

示例会展示选图、拍照、保存和图片尺寸反馈，用户取消时页面保持可操作。

#什么时候用

  • 从照片库选择一张图，传给 AppUI、Vision、Core ML 或上传流程。
  • 使用相机拍照并保存到照片库。
  • 把下载、生成或导出后的本地视频保存到系统照片库。
  • 读取资源元数据、相册列表、收藏、最近图片或图片尺寸。
  • 把图片转成 Base64，交给 vision_helper、网络接口或 JSON 数据。

#核心模型

pick_image() 默认返回 PIL.Image；如果环境没有 Pillow，或者你需要原始 bytes，使用 pick_image(raw_data=True)。capture_image() 返回拍照后的图片对象；save_image(...) 支持 PIL.Image 或 bytes。

save_video(path, ...) 只接收本地视频文件路径，例如 network.download(...) 写入的 mp4 文件。视频不会读入 Python 内存，album_name 等兼容参数当前会被接受但忽略。

资源接口使用 Asset 和 AssetCollection。读取大量资源时必须传 limit，不要一次性加载全部图片或图片数据。

#AppUI 中怎么接

选图和拍照必须在按钮回调里执行。下面的示例只展示尺寸和状态，不在页面构建时访问照片库。

#保存视频

下载或生成视频后，把本地文件路径传给 save_video(...)。不要把视频 bytes 读入内存再传入。

save_video(...) 返回 True 或 False。常见失败原因包括文件不存在、照片写入权限被拒、视频格式不被系统照片库支持，或文件过大导致系统处理耗时较长。

#读取资源

media_type 支持 photo、video、all。limit=0 表示不限制，列表页不建议这样使用。

#Asset 对象

常用属性：

  • local_identifier
  • media_type
  • pixel_width
  • pixel_height
  • creation_date
  • modification_date
  • location
  • duration
  • is_favorite

#Base64 与视觉识别

vision_helper 的检测 API 接收 Base64 字符串，vision 的 OCR API 接收 bytes。

#失败路径

情况应该怎么处理
用户取消选图或拍照返回 None，保持原状态并显示取消。
没有 Pillow使用 raw_data=True 或 Base64 API。
照片权限被拒绝显示授权说明，不循环打开选择器。
图片很大读取尺寸或缩略信息，不把 bytes/Base64 放进 storage。
视频文件不存在或格式不支持先确认下载/导出成功，再把本地路径传给 save_video(...)。

#常见错误

错误写法后果修正
不判断 pick_image() 返回值用户取消后代码崩溃先判断 if image is None。
列表页 get_assets(limit=0)可能一次性加载过多设置合理 limit。
把 Base64 图片存进 storage占用大且慢存文件路径或重新选择。
把图片路径传给 vision_helper输入类型错误先用 pick_image_base64 或 image_to_base64。
