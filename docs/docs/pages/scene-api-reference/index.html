scene API 参考

集中列出 scene 生命周期、节点、Action、Physics 与 Classic 绘图签名。

scene 用来做 2D 场景、节点、动画、触摸、物理和 Classic 绘图。它适合小游戏、可视化、画布、手势动画；普通表单、列表和设置页不要用 scene。

#Scene 生命周期

方法触发时机
setup()场景第一次显示前
update()每帧更新逻辑
draw()Classic 绘图模式每帧绘制
touch_began(touch)触摸开始
touch_moved(touch)触摸移动
touch_ended(touch)触摸结束
did_change_size()尺寸或方向变化
pause()App 进入后台
resume()App 回到前台
stop()场景关闭

#节点模式

节点模式适合游戏对象、精灵、文字、粒子和动作动画。

#常用节点

API用途
Node基础节点，管理层级、位置、缩放、透明度
SpriteNode图片或纯色矩形精灵
LabelNode文字节点
ShapeNode路径和形状
EffectNode后处理效果
EmitterNode粒子效果
Texture位图纹理
Action移动、缩放、旋转、淡入淡出、序列、循环

#Classic 绘图

Classic 绘图适合简单画布、教学示意和轻量图形。

API用途
background(r, g, b)设置背景
fill(r, g, b, a=1)设置填充色
stroke(r, g, b, a=1)设置描边色
rect(x, y, w, h, corner_radius=0)画矩形
ellipse(x, y, w, h)画椭圆
line(x1, y1, x2, y2)画线
image(name, x, y, w, h)画图片
text(txt, x=..., y=...)画文字
push_matrix() / pop_matrix()保存和恢复变换

#物理

API用途

#失败路径

情况应该怎么处理
场景不显示确认最后调用 scene.run(SceneSubclass())，并且 setup() 没有抛错。
触摸无反应使用 touch_began / touch_moved / touch_ended，不要把事件逻辑写进普通函数后忘记绑定。
帧率下降不要在 update() 或 draw() 中做网络请求、文件大读写或反复创建大量节点。
普通表单很难做表单、设置、列表和导航页改用 appui。

#使用规则

  • 游戏、精灵、连续绘制、物理动画用 scene。
  • 表单、设置、列表、导航不要用 scene，用 appui。
  • 每帧 update() / draw() 中不要做网络请求、文件大读写或阻塞等待。
  • 触摸处理里只更新状态或启动 Action，重任务放到后台。
