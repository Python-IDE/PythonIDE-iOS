vision_helper

人脸、条码、矩形与图像分类。

人脸、条码、矩形与图像分类。

#预期效果

示例会展示 Base64 图片如何进入人脸、条码、矩形或分类检测流程。

#适用场景

Vision 辅助接口，覆盖人脸、条码、图像分类、矩形检测和异步轮询。

#标准示例

#AppUI 图片检测配方

需要把图片分类或条码检测放进页面时，先用 photos.pick_image_base64(...) 得到 Base64 字符串，再调用 vision_helper。示例提供分类和条码两个按钮。

预期效果：用户每次点击后选择图片；分类或条码检测结果显示在页面中，取消和不可用状态不会抛出异常。

#API 参考

类型API签名说明
functionsetup_vision_frameworksetup_vision_framework() -> Any设置Vision框架 - 支持多种导入方式 返回Vision框架的类，如果不可用则返回None
functionis_vision_availableis_vision_available() -> bool检查Vision框架是否可用
functiondetect_facesdetect_faces(image_base64: Any) -> AnyDetect faces in a base64-encoded image.
functiondetect_face_landmarksdetect_face_landmarks(image_base64: Any) -> AnyDetect face landmarks (eyes, nose, lips) in a base64-encoded image.
functiondetect_barcodesdetect_barcodes(image_base64: Any) -> AnyDetect barcodes/QR codes in a base64-encoded image.
functionclassify_imageclassify_image(image_base64: Any, max_results: int=...) -> AnyClassify an image (iOS 15+).
functiondetect_rectanglesdetect_rectangles(image_base64: Any) -> AnyDetect rectangles in a base64-encoded image.
functionasync_requestasync_request(operation: Any, image_base64: Any, callback_id: Any, extra: int=...) -> AnyStart an async vision operation. Poll with poll_result(callback_id).
functionpoll_resultpoll_result(callback_id: Any) -> AnyPoll for an async vision result. Returns None if not ready, or a result dict.

#失败路径

情况应该怎么处理
权限被拒绝显示当前状态和授权说明，不要循环弹出系统授权。
用户取消系统面板处理 None、False、空列表或错误字典，保持页面可继续操作。
设备或系统版本不支持显示不可用状态，并提供替代流程或禁用对应按钮。
返回数据为空把空状态写回界面，不把空结果当作异常崩溃。

#使用规则

  • vision_helper 接收 Base64 图片字符串。
  • 耗时识别可用 async_request + poll_result。
