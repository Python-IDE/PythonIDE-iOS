live_activity

锁屏和 Dynamic Island 的 Live Activity 控制。

live_activity 用来在锁屏和 Dynamic Island 显示一个正在进行的任务，例如上传、导出、计时、训练或配送状态。它不是普通页面提示，也不是长期后台任务机制。

#预期效果

示例会展示 Live Activity 的启动、进度更新、结束和不可用状态处理。

#什么时候用

  • 一个任务需要在离开 App 后继续显示进度。
  • 任务有明确开始、更新和结束。
  • 进度或状态适合展示在锁屏，不包含敏感内容。

#核心模型

先用 is_supported() 判断设备和系统是否支持。start(...) 开始活动，update(...) 更新标题、消息、进度、图标和紧凑文字，end(...) 结束活动并可设置延迟移除时间。

progress 应使用 0.0 到 1.0 的范围。无论成功、失败还是取消，都要调用 end(...)，避免锁屏状态残留。

#失败路径

情况应该怎么处理
is_supported() 为 False隐藏或禁用入口，改用页面状态或通知。
任务失败调用 end(message="Failed")，同时在 App 内显示失败。
用户取消立即 end(...)，不要留下旧进度。
任务没有明确结束不适合 Live Activity。

#常见错误

错误写法后果修正
普通按钮反馈也启动 Live Activity打扰用户锁屏用 AppUI 文本、HUD 或本地通知。
更新进度超过 1.0显示异常限制在 0.0 到 1.0。
只 start 不 end锁屏状态残留在成功、失败、取消路径都调用 end。

#API 参考

类型API签名说明

functionstartstart(title="", message="", progress=None, icon=None, compact_text=None) -> Any开始一个 Live Activity。

functionendend(message=None, dismiss_delay=None) -> Any结束当前 Live Activity。
