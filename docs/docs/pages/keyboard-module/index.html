keyboard

编辑器自定义键盘工具栏与插入动作。

编辑器自定义键盘工具栏与插入动作。

#预期效果

示例会展示编辑器键盘工具栏按钮、插入文本和恢复默认按钮。

#适用场景

给内置代码编辑器添加自定义键盘工具栏按钮和片段动作。

#标准示例

#API 参考

类型API签名说明
functionadd_buttonadd_button(label: Any, action: Optional[Callable[..., Any]]=..., icon: Optional[Any]=...) -> Any添加一个按钮到键盘工具栏。
functionadd_groupadd_group(name: Any, buttons: Any) -> Any添加一组按钮。
functionremove_buttonremove_button(label: Any) -> None移除指定按钮
functionclearclear() -> Any清空所有自定义键盘按钮
functioninsert_textinsert_text(text: Any, offset: int=...) -> Any在编辑器当前光标位置插入文本。
functioninsert_snippetinsert_snippet(template: Any) -> Any插入代码片段模板（支持 $1, $2, $0 占位符）。
functionmove_cursormove_cursor(offset: Any) -> Any移动光标位置。
classButtonButton(label: Any, action: Optional[Callable[..., Any]]=..., icon: Optional[Any]=...) -> None键盘工具栏按钮。
functionset_buttonsset_buttons(buttons: Any) -> None替换所有键盘工具栏按钮。

#使用规则

  • keyboard 只影响代码编辑器工具栏，不影响 appui TextField。
  • 按钮 action 用命名函数；复杂片段用 insert_snippet。
  • 替换整套工具栏前调用 clear() 或 set_buttons(...)，避免重复按钮。
  • 片段占位符使用 $1、$2、$0，不要写 ${cursor}。

#AppUI 之外的按钮

keyboard 是编辑器键盘工具栏，不是 AppUI 页面按钮。它的回调会在编辑器里插入文字或移动光标。

#失败路径

情况应该怎么处理
按钮没有反应检查 action 是否是命名零参数函数。
按钮重复出现先 clear() 或改用 set_buttons(...)。
光标位置不对用 $0 或 offset 控制最终位置。
需要页面 UI使用 AppUI，不要用 keyboard。
