c_extensions

内置扩展库清单、可用范围和替代建议。

内置扩展库清单、可用范围和替代建议。

#预期效果

示例会展示如何查看内置扩展可用性，并为不可用模块选择替代方案。

#适用场景

内置 C 扩展库清单和替代建议；它不是可导入模块。

#核心模型

c_extensions 是文档页，不是 Python 模块。不要写 import c_extensions。当一个第三方包包含 C/Rust/native 扩展时，只有已经内置或能被当前环境安装的包才能使用；未内置的大型科学计算、CV、深度学习包通常不能直接安装。

优先选择已内置包、纯 Python 替代，或 PythonIDE 已封装的原生模块。

#先选路线

需求优先路线
数组、表格、统计、简单模型先试 numpy / pandas。
压缩、哈希、二进制序列化使用已内置的 zstandard、xxhash、msgpack、msgspec。
OCR、条码、人脸和图片分类使用 vision / vision_helper。
iOS 设备能力使用对应原生模块，不安装桌面平台包。
只差一个小工具函数先找纯 Python 包，或直接写小型实现。
依赖未内置大型 native 包明确不可直接使用，并提供替代工作流。

#使用规则

  • c_extensions 是文档页，不是 import 名。
  • 需要未内置的 C 扩展时，优先换纯 Python 包或内置替代。
  • 不要把桌面 Linux/macOS 的 pip 成功经验直接套到 iOS 运行时。
  • 示例里要先 try/except ImportError，给出降级路径。

#失败路径

情况应该怎么处理
ImportError检查是否内置，提供纯 Python 或内置替代。
包名和 import 名不同使用真实 import 名。
用户需要未内置包说明不可直接使用，并给替代路线。
性能不足先尝试 numpy / pandas 等已内置包。

#发布前检查

检查项合格标准
import示例使用真实包名，不写 import c_extensions。
降级ImportError 有纯 Python 或内置模块替代。
平台没有承诺桌面 pip 包能直接在 iOS 运行。
