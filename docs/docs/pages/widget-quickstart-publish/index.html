widget / 小组件

从脚本到桌面

写脚本、预览、调参数、发布到桌面并排错。

这篇教程给出一条完整路径：写最小脚本、预览、调整参数、发布到桌面，再处理常见失败。

适合第一次发布小组件时使用。已经熟悉流程后，可以直接查 布局与尺寸、参数面板、状态和交互 或 API 地图。

#完整示例

预期效果：medium 尺寸显示标题、主数值、进度条、目标说明和按钮；small 尺寸保留标题、主数值、进度条和按钮，隐藏次要说明以避免拥挤。

#发布流程

  1. 运行脚本，打开小组件预览。
  2. 在预览面板调整 标题、强调色 和 目标次数。
  3. 确认 small、medium 或目标 family 内没有文字裁切。
  4. 发布到桌面。
  5. 在桌面点击按钮，确认数值变化并刷新。

如果改了参数名、状态 key 或布局结构，重新运行并发布。桌面仍显示旧内容时，删除桌面小组件后重新添加。

#发布前检查

检查项合格标准
入口只创建一个 Widget()，最后调用 w.render()
文本用户可改文本、按钮和主值使用 .line_limit(1).min_scale(...)
尺寸small 只保留核心信息，medium/large 再增加辅助内容
参数widget.param.* 在构建 UI 前声明，默认值类型正确
状态按钮和开关使用 widget.state 生成的 action
外观浅色/深色、透明/染色模式下仍然可读

#失败路径

  • 预览空白：检查 w.render()，并确认脚本没有混用 AppUI、网页框架或普通 App 页面代码。
  • 文本裁切：先缩短文字，再加 .line_limit(1).min_scale(0.6)；仍然拥挤时按 family 隐藏次要内容。
  • 参数不显示：确认使用的是 widget.param.text/color/slider/number/bool/choice/file，并且声明在 w.render() 之前。
  • 点击没反应：确认 action 来自 widget.state，例如 count.increment() 或 done.toggle()。
  • 桌面不刷新：重新运行并发布；仍旧时删除桌面小组件后重新添加。
