演示 notification 请求权限与 schedule，并用 storage 持久化表单偏好。

#预期效果

运行后会出现通知提醒表单，权限、延迟分钟、本地提醒和取消操作都有可见反馈。

#关键技巧

  • 先 request_permission()，再用稳定 identifier 调用 schedule / remove_pending 便于覆盖或取消。
  • request_permission() 返回 dict，无需 json.loads。
  • 用户偏好用 storage.get_json / set_json 在 on_appear 与 on_change 中同步。
