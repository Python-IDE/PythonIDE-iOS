iOS 原生模块 / 系统服务

permission

统一查询权限状态，并请求位置或通知权限。

permission 是统一权限状态入口，用来查询和请求相机、麦克风、照片、位置、通讯录、日历、提醒事项、通知、语音、运动、蓝牙和健康权限。它只处理权限本身，真正读取照片、定位、通知调度等业务仍然调用对应模块。

#预期效果

示例会展示统一查询权限、请求权限，并把授权、拒绝和不可用状态写回界面。

#什么时候用

  • 设置页里展示所有关键权限状态。
  • 在执行 location、photos、contacts 等功能前做前置判断。
  • 用同一套返回结构处理授权、拒绝、不可用和调用失败。

#核心模型

status(module) 查询权限，request(module) 弹出系统授权，status_all() 返回所有支持模块的状态。request() 会等待用户响应，因此应该放在按钮、菜单或明确动作里，不要放在 AppUI body() 中。

合法模块名来自 MODULES：

当前模块名包括 camera、microphone、photos、location、location_always、contacts、calendar、reminders、notifications、speech、motion、bluetooth、health。通知权限的键是 notifications，不是 notification。

#AppUI 权限流程

权限请求应该由用户动作触发。被拒绝后不要反复弹窗，改为展示状态和引导说明。

#失败路径

情况应该怎么处理
用户拒绝授权展示当前状态和用途说明，不要循环请求。
模块名写错返回错误或无效状态
需要具体业务数据调用对应业务模块，例如 photos.pick_image()。
能力不可用显示失败状态，保留页面可操作。

#常见错误

错误写法后果修正
permission.request("notification")模块名不匹配使用 permission.request("notifications")。
在页面构建时请求权限页面刷新时反复弹系统授权只在按钮回调里请求。
只查权限不处理拒绝用户不知道下一步显示状态和说明。
