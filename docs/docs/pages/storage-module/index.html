storage

UserDefaults 持久键值存储。

storage 是轻量级 UserDefaults 键值存储，适合保存设置项、开关、上次选择、体积很小的 JSON 配置。它不是文件数据库，也不是敏感数据存储。

#预期效果

示例会展示 UserDefaults 风格键值读写，并在 AppUI 表单中同步保存状态。

#什么时候用

  • 保存主题、排序方式、筛选条件、开关状态。
  • 保存小型 JSON，例如用户偏好、最近打开记录、表单草稿。
  • 在 AppUI、Widget、Shortcuts 相关脚本里共享简单配置。

不要把密码、token、私钥放进 storage；这类数据用 keychain。可查询的大列表、播放历史、收藏和离线缓存用 database。

#API 参考

API返回用途
get(key, default=None)Any读取字符串或底层保存的值；不存在时返回 default。
set(key, value)None写入字符串或可被 UserDefaults 接受的简单值。
get_int(key, default=0) / set_int(key, value)int / None保存计数、索引、步进值。
get_float(key, default=0.0) / set_float(key, value)float / None保存音量、比例、阈值。
get_bool(key, default=False) / set_bool(key, value)bool / None保存开关状态。
get_json(key, default=None) / set_json(key, value)Any / None保存 JSON 可序列化对象。
remove(key)None删除一个键。
has_key(key)bool判断键是否存在。
all_keys(prefix="")list[str]列出所有键；传 prefix 可以按命名空间筛选。

#清理命名空间

如果一个页面有多项设置，用统一前缀管理，后续清理不会误删其他脚本的数据。

#失败路径

情况应该怎么处理
首次运行没有键读取时传 default，不要让界面显示 None。
JSON 结构升级读取后补默认字段，再保存新结构。
用户退出账号删除账号相关前缀键，保留通用偏好。
数据变大或包含图片改用 database、文件系统或媒体模块。

#常见错误

错误写法后果修正
把 token 放进 storage敏感数据不安全使用 keychain。
多个脚本共用 theme 这种短键键名互相覆盖使用 settings.theme 这类命名空间。
不传默认值首次运行逻辑容易失败使用 storage.get("key", default)。

#使用规则

  • 键名加命名空间，例如 settings.theme、todo.filters，避免不同脚本互相覆盖。
  • JSON 保持小而稳定；大列表、播放历史、收藏、缓存请用 database，大文件、图片、日志请写文件。
  • 读取时总是传默认值，让首次运行和清空设置后仍能工作。
  • 删除一组键时先用 all_keys(prefix) 找到范围，再逐个 remove。
