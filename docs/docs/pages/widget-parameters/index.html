widget / 小组件

参数面板

让用户在预览面板调颜色、字号、开关、选项和文件。

凡是希望用户在预览面板里调节的值，都用 widget.param 声明。参数是构建时输入，返回值可以直接传给 Widget()、text()、progress()、shape() 等 API。

#常用参数

预期效果：预览面板显示标题、背景色、文字色、标题字号、紧凑模式和风格参数；标题和说明都不会超出卡片。

API用途
widget.param.text(name, default)标题、备注、文件名
widget.param.color(name, default)背景色、文字色、线条色
widget.param.slider(name, default, min=..., max=..., step=...)字号、线宽、透明度、圆角
widget.param.number(name, default, min=..., max=...)阈值、目标值、数量
widget.param.bool(name, default)是否显示、是否紧凑
widget.param.choice(name, options, default=...)主题、模式、数据范围
widget.param.file(name, default, extensions=[...])SVG、背景图、文本数据

#参数和桌面显示

参数属于小组件配置，不是桌面点击状态。预览面板修改参数后会重新构建当前预览；发布到桌面后，桌面小组件使用发布时的参数配置。改过参数名、默认值或脚本结构后，需要重新运行并发布。

#参数命名

参数名应该是用户能看懂的中文，例如 "背景色"、"标题字号"、"数据文件"。不要把代码里的临时变量名直接暴露给用户，除非它本来就是文件路径或排查字段。

#失败路径

  • 参数不出现在预览面板：确认用的是 widget.param.*(...)，并且声明发生在 w.render() 之前。
  • 默认值类型不匹配：slider 和 number 用数字，bool 用布尔值，choice 的 default 必须在 options 里。
  • 文件参数找不到资源：把默认文件放在脚本同目录或 assets/ 目录；需要用户选择时用 widget.param.file(...)。
  • 桌面没有同步新参数：重新运行脚本并发布；改过参数名时，旧桌面实例可能需要重新添加。

#API 参考

参数完整签名见 API 参考：重点查 widget.param.text、color、slider、number、bool、choice 和 file。
