iOS 原生模块 / 系统服务

dialogs

原生对话框、表单和日期选择。

dialogs 提供阻塞式原生对话框、输入框、列表选择、简单表单和日期选择。它适合短脚本里的临时交互；复杂页面、可编辑设置和多步骤流程优先用 AppUI。

#预期效果

示例会展示 alert、confirm、输入框和表单对话框的返回值处理。

#什么时候用

  • 脚本开始前询问一个名称、模式或日期。
  • 执行危险操作前弹一次确认。
  • 用少量字段快速收集配置，然后继续运行脚本。

#核心模型

dialogs 调用会阻塞当前脚本直到用户响应。返回值可能是按钮序号、字符串、字典、列表或空值，必须处理取消和空输入。AppUI 页面中不要滥用阻塞对话框；页面内交互优先使用 .sheet、.alert、Form 或原生控件。

#选择方式

需求API
简单确认alert
单个文本输入input_alert
从几个选项中选择list_dialog
一次收集少量字段form_dialog
日期或时间date_dialog
短暂成功提示hud_alert

#失败路径

情况应该怎么处理
用户取消检查返回值，停止后续危险操作。
输入为空给默认值或提示重新输入。
多选列表返回空列表保持原选择，不写入配置。
需要持续编辑改用 AppUI Form。

#常见错误

错误写法后果修正
不处理取消返回值后续代码用空值报错每次调用后先判断返回值。
用 form_dialog 做复杂动态表单体验和状态难维护用 AppUI。
在 AppUI 回调里连续弹多个阻塞对话框页面卡顿合并流程或用 sheet。
