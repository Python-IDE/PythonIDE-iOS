widget / 小组件

widget

用 Python 编写主屏、锁屏和 StandBy 小组件。

widget 是 PythonIDE 的原生 WidgetKit 小组件 API。基本流程是：创建 Widget()，添加内容、布局、参数、状态和时间线，最后调用 w.render()。

脚本只描述小组件的结构和行为意图；渲染、系统外观、桌面刷新和 AppIntent 执行由 PythonIDE 与 WidgetKit 处理。小组件脚本不要混用 AppUI、网页框架或普通 App 页面模型。

#最小脚本

预期效果：medium 尺寸显示标题、主数值、进度条和按钮；数值变化时使用系统数字过渡，按钮标题不会挤出边界。

这段代码展示了实际脚本里最常见的四个约定：

  • widget.param.color(...)：在预览面板里出现可调颜色。
  • widget.state.int(...)：桌面小组件点击按钮后写入状态。
  • .content_transition("numericText")：数字变化时使用 WidgetKit 数据更新动画。
  • .line_limit(1).min_scale(...)：所有可变文本默认防止小尺寸裁切。

#先选写法

大部分小组件只需要 row()、column()、grid()。只有表格、键盘、Bingo、翻页钟、海报式排版这类精确作品，才需要 table()、canvas()、path() 和 .place()。

#写小组件的心智模型

  1. 根容器：w = widget.Widget(...)，只管整体背景、边距和系统外观。
  2. 内容节点：text()、value()、symbol()、image()、progress()、line_chart()。
  3. 布局节点：row()、column()、layer()、grid()、table()、canvas()。
  4. 参数：widget.param 声明用户可调的值。
  5. 状态：widget.state 声明桌面点击能修改的数据。
  6. 时间线：w.timeline(...) 声明未来多个显示结果。
  7. 结尾：永远调用 w.render()。

#失败路径

  • 预览是空白：先确认脚本最后有 w.render()，并且只创建一个 Widget()。
  • 小尺寸拥挤：不要缩小整张 large 卡片，去 布局与尺寸 按 family 分支。
  • 参数不出现：参数必须在构建 UI 前用 widget.param 声明，详情见 参数面板。
  • 点击没反应：按钮或开关必须绑定 widget.state 生成的 action，详情见 状态和交互。
  • 桌面仍显示旧内容：重新运行并发布；必要时删除桌面小组件后重新添加，更多见 排错。

#WidgetKit 的边界

Widget 不是普通 App 页面。它不能做文本输入、滚动长列表、WebView、视频、复杂手势或 60fps 实时动画。它擅长的是：漂亮的静态展示、系统允许的数据更新动画、AppIntent 按钮/开关、深色/透明/染色外观、时间线刷新和 Live Activity/Control 的安全入口。
