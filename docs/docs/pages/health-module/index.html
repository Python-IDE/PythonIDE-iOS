iOS 原生模块 / 设备与传感器

health

HealthKit 授权、查询和体重写入。

health 用来请求 HealthKit 授权、读取步数、心率、体重、锻炼、睡眠和血压数据，也可以写入体重样本。健康数据非常敏感，界面和存储都应该只保留当前功能真正需要的内容。

#预期效果

示例会展示 HealthKit 授权、步数/心率等数据读取和空结果处理。

#什么时候用

  • 做步数、心率、睡眠、锻炼或体重仪表盘。
  • 读取并汇总最近一段时间的血压记录。
  • 保存用户主动输入的体重样本到 HealthKit。

#核心模型

先用 is_available() 判断设备是否支持 HealthKit，再用 request_authorization(read=..., write=...) 请求最小权限。读取时间范围使用 Unix 时间戳；不传 start、end 时，不同查询会使用模块默认范围，例如步数默认最近 24 小时，血压默认最近 30 天。

授权类型名应使用模块支持的 HealthKit 简写，例如 steps、heart_rate、weight、sleep、blood_pressure、blood_pressure_systolic、blood_pressure_diastolic。不要写成 step_count。

授权和读取都应该放在按钮或刷新回调里。下面的示例只展示步数，不在页面构建时读取 HealthKit。

#健康仪表盘配方

需要同时展示步数、心率和睡眠时，先请求最小读取集合，再把空结果写成明确状态。示例不保存健康明细，只展示当前页面需要的汇总。

预期效果：点击按钮后才请求 HealthKit 读取权限；授权成功显示步数和记录数量，拒绝或不可用时页面保留清晰状态。

#失败路径

情况应该怎么处理
is_available() 为 False显示设备不支持，不展示空白图表。
用户拒绝授权说明需要哪类数据，不要循环弹授权。
查询结果为空显示空状态，例如“最近没有记录”。
血压或心率异常只做数据展示，不在文档和示例中给诊断结论。

#常见错误

错误写法后果修正
read=["step_count"]类型名不匹配使用 read=["steps"]。
一次请求所有健康权限用户更容易拒绝只请求当前页面需要的类型。
把健康明细写入普通键值存储敏感数据扩散只保存必要汇总，或让 HealthKit 作为来源。

#API 参考

类型API签名说明
functionis_availableis_available() -> bool当前设备是否支持 HealthKit。
functionrequest_authorizationrequest_authorization(read=None, write=None) -> Any请求 HealthKit 读写权限；read、write 是类型名列表。
functionauthorization_statusauthorization_status(type_id) -> Any查询某个类型的授权状态。
functionquery_stepsquery_steps(start=None, end=None) -> Any查询步数，默认最近 24 小时。
functionquery_heart_ratequery_heart_rate(start=None, end=None) -> Any查询心率列表，记录包含 value、start、end。
functionquery_weightquery_weight(limit=10) -> Any查询最近体重记录，单位为 kg。
functionsave_weightsave_weight(kg, timestamp=None) -> Any保存体重样本到 HealthKit。
functionquery_workoutsquery_workouts(limit=20) -> Any查询最近锻炼记录。
functionquery_sleepquery_sleep(start=None, end=None) -> Any查询睡眠分析数据，默认最近 7 天。
functionquery_quantityquery_quantity(type_id, start=None, end=None, unit="count") -> Any通用数量查询，例如 steps、heart_rate、distance。
functionquery_blood_pressurequery_blood_pressure(start=None, end=None) -> Any查询配对血压记录，默认最近 30 天，单位为 mmHg。
functionsummarize_blood_pressuresummarize_blood_pressure(records=None, start=None, end=None) -> Any返回血压数量、最近值、平均值、最小值和最大值；不做诊断。
