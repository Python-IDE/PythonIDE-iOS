sound

音效播放、长音频播放器和静音开关行为。

音效播放、长音频播放器和静音开关行为。

#预期效果

示例会展示短音效、长音频播放器、音量滑块和播放状态反馈。

#适用场景

短音效播放、长音频 Player、MIDI Player 和音量控制。短反馈用 play_effect，完整音频文件用 Player，不要混用。

#核心模型

play_effect(...) 适合短反馈音，返回的 handle 可以传给 stop_effect(handle)。如果 looping=True，必须保存 handle，否则之后无法停止指定音效。

Player(file_path) 适合本地长音频文件，提供 play()、pause()、stop()、current_time、duration、volume、rate 和 number_of_loops。远程视频或单个 URL 播放优先看 avplayer。

#API 参考

类型API签名说明
functionplay_effectplay_effect(name: str, volume=1.0, pitch=1.0, pan=0.0, looping=False) -> Any播放短音效，返回可停止的 handle。
functionstop_effectstop_effect(effect_handle) -> None停止某个短音效。
functionstop_all_effectsstop_all_effects() -> None停止所有短音效。
classPlayerPlayer(file_path="") -> None本地长音频播放器。
classMIDIPlayerMIDIPlayer(file_path="", sound_bank_path="") -> NoneMIDI 播放兼容接口。
functionset_volumeset_volume(volume: float) -> None设置全局主音量，范围 0.0 到 1.0。
functionset_honors_silent_switchset_honors_silent_switch(flag: bool) -> None设置是否遵守 iOS 静音开关。
functionget_volumeget_volume() -> float获取当前全局主音量。

#使用规则

  • 短提示音用 play_effect；长音频用 Player。
  • 页面关闭或任务结束时停止循环音效。
  • 循环音效必须保存 handle，后续用 stop_effect(handle) 结束。
  • 全局音量影响所有效果和播放器，设置页里要给用户明确反馈。
  • 需要遵守静音开关时设置 set_honors_silent_switch(True)。

#选择方式

需求API
点击提示、成功音、错误音play_effect
停止某个短音效stop_effect(handle)
停止所有短音效stop_all_effects()
播放本地长音频Player
MIDI 文件兼容MIDIPlayer

#失败路径

情况应该怎么处理
play_effect 返回空显示音效不可用或文件不存在。
循环音效需要停止保存 handle 并调用 stop_effect(handle)。
用户离开页面停止循环音效或播放器。
需要远程视频播放使用 avplayer。
