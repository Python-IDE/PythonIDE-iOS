iOS 原生模块 / 设备与传感器

biometric

Face ID / Touch ID 验证。

biometric 用来调用 Face ID、Touch ID 或设备密码回退完成本机身份确认。它只能证明当前设备用户通过了系统认证，不能替代服务器登录、权限模型或敏感数据存储。

#预期效果

示例会展示 Face ID / Touch ID 验证结果，并把成功、取消或失败原因写回界面。

#什么时候用

  • 打开本地安全笔记、API Key、隐私设置前做二次确认。
  • 让用户在 Face ID / Touch ID 不可用时使用设备密码回退。
  • 检查当前设备支持的是 face_id、touch_id 还是 none。

#核心模型

is_available() 只说明设备当前能否使用生物认证。authenticate(reason) 只尝试生物认证；authenticate_with_passcode(reason) 允许系统密码作为回退，适合更稳的解锁流程。

认证结果是字典，成功时通常包含 success: True，取消、失败、能力不可用时会包含 error。不要把返回值当成布尔值直接用，也不要把敏感明文打印到日志。

#失败路径

情况应该怎么处理
biometric_type() 返回 none展示不可用状态，提供普通登录或说明。
用户取消保持锁定状态，不反复弹认证框。
生物认证失败可改用 authenticate_with_passcode 提供设备密码回退。
需要保存 secret使用 keychain，不要放进 storage。

类型API签名说明
