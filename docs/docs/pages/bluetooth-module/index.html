BLE 扫描、连接、服务发现和特征读写。

#预期效果

示例会展示蓝牙可用性检查、扫描、连接和设备不可用时的提示。

#适用场景

BLE 状态、扫描、连接、服务发现和特征读写。它面向低功耗蓝牙外设，不是普通网络连接；HTTP 和 WebSocket 请使用对应网络模块。

#AppUI 扫描面板

BLE 流程通常需要多次用户动作：先扫描，再选择或连接，再发现服务。示例只连接扫描结果中的第一个设备，真实产品应该让用户选择目标外设。

预期效果：页面不会自动扫描；点击扫描后显示设备数量，点击连接后展示服务数量，失败时保留错误字符串，断开按钮会释放连接。

#API 参考

类型API签名说明
functionstatestate() -> Any返回 BLE 状态，例如 powered_on、powered_off、unauthorized。
functionscanscan(duration: float=5.0) -> Any扫描 BLE 外设，返回包含 uuid、name、rssi 的字典列表。
functionstop_scanstop_scan() -> None立即停止当前 BLE 扫描。
functionconnectconnect(uuid: Any) -> Any连接外设，成功返回 ok，失败返回错误字符串。
functiondisconnectdisconnect(uuid: Any) -> Any断开指定外设连接并释放会话状态。
functiondiscover_servicesdiscover_services(uuid: Any) -> Any发现已连接外设的服务和特征。
functionreadread(peripheral_uuid, service_uuid, characteristic_uuid) -> Any读取特征值，返回 Base64 字符串。
functionwritewrite(peripheral_uuid, service_uuid, characteristic_uuid, base64_data) -> Any写入 Base64 编码数据，成功返回 True。

#使用规则

  • 扫描会消耗电量，设置短 duration 并在不需要时 stop_scan。
  • 写入特征值使用 base64 数据。
  • 先检查 state()，再扫描或连接。
  • 连接、读取、写入都要处理失败字符串或空结果。
  • 特征值是二进制数据时，用 base64 编码传入 write。

#BLE 操作顺序

  1. state() 确认可用。
  2. scan(duration=...) 找到目标外设。
  3. connect(uuid) 建立连接。
  4. discover_services(uuid) 找到服务和特征。
  5. read(...) 或 write(...) 进行数据交换。
  6. disconnect(uuid) 释放连接。

#失败路径

情况应该怎么处理
state() 不是 powered_on显示蓝牙未开启或未授权，不继续扫描。
扫描返回空列表提示未发现设备，并允许重试。
connect(...) 不是 ok显示错误字符串，不继续读写。
read(...) 返回空字符串视为读取失败或无数据。

#常见错误

错误写法后果修正
判断 poweredOn状态字符串不匹配使用 powered_on。
直接写 bytesAPI 需要 Base64 字符串先 base64.b64encode(...)。
忘记 disconnect连接资源残留用 try/finally 释放。
