原生 WebSocket 连接、收发和回调。

#预期效果

示例会展示 WebSocket 连接、发送、接收、关闭和超时处理。

#适用场景

原生 WebSocket 连接、同步收发和事件回调。适合聊天、行情、设备状态流这类需要保持连接的任务；普通一次性请求优先使用 network。

#事件回调示例

回调适合由连接主动推送消息的场景。回调函数保持轻量，把复杂解析放到普通函数里。

#AppUI 中怎么接

连接对象要保存在 State 之外或专门的持有变量里，并提供关闭入口。下面的示例把连接、发送、关闭都放在按钮回调中。

#API 参考

类型API签名说明
functionconnectconnect(url: Any, protocols: Optional[Any]=...) -> Any连接到 WebSocket 服务，返回连接句柄。
classWebSocketWebSocket(conn_id: Any) -> NoneWebSocket 连接句柄。

#WebSocket 实例方法

方法用途
state当前连接状态：open、connecting、closing、closed。
is_open当前是否为打开状态。
send(data)发送字符串或可序列化数据。
receive(timeout=30)同步等待下一条消息，超时返回空值。
close(code=1000)主动关闭连接。
on_open / on_message / on_error / on_close设置事件回调。

#失败路径

情况应该怎么处理
连接失败捕获异常，显示服务器或网络错误。
receive(...) 超时显示暂无消息，不阻塞界面。
断线或关闭更新状态，禁止继续发送。
页面退出调用 close()。

#使用规则

  • 连接结束时调用 close。
  • 回调函数保持轻量，耗时工作放后台。
  • 处理断线和超时。
  • 在 AppUI 页面中不要在 body() 里连接；用按钮、刷新或页面初始化逻辑触发，并把状态写回 State。
