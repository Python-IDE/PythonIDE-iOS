widget / 小组件

资源与外观

深色、透明、染色、背景图、SVG 和 SF Symbol。

WidgetKit 负责浅色、深色、透明、染色等系统外观。脚本只声明外观意图；不要用手写遮罩去模拟系统背景策略。

#背景

预期效果：文本保持原色，图标参与系统染色；背景可交给 WidgetKit 在支持的场景中移除。

建议：

  • 普通背景用 Widget(background=...)。
  • 系统 Widget 背景用 w.container_background(...)。
  • 透明模式用 w.transparent_background(True) 或 removable=True。
  • 染色模式中，参与系统染色的节点用 .accentable(True)；保持原色用 .accentable(False)。

#SF Symbol

预期效果：SF Symbol 使用 palette 渲染，标题在图标下方保持单行。

#SVG

预期效果：SVG 图标按指定尺寸和颜色显示，标题在不同尺寸下保持可读。

SVG 可以放在脚本同目录的 assets/ 文件夹里，也可以用文件参数让用户选择。

#图片背景

预期效果：浅色和深色模式使用不同背景图，底部遮罩保证白色标题可读。

深色图片应该单独提供，不要只把浅色图加黑色遮罩。

#失败路径

  • 染色模式下颜色不对：系统只会处理 .accentable(True) 的节点；必须保留原色的节点显式写 .accentable(False)。
  • 透明背景不生效：同时检查 w.container_background(..., removable=True) 和 w.transparent_background(True)，桌面宿主是否允许透明由系统决定。
  • 图片在深色模式发灰：为 background_image((light, dark), ...) 分别提供浅色和深色图，不要只靠遮罩模拟。
  • SVG 不显示：确认文件路径在脚本目录或 assets/ 目录，或者改用 widget.param.file(...) 让用户选择。

#API 参考

外观和资源完整签名见 API 参考：重点查 container_background、content_margins、transparent_background、background_image、w.image、w.svg、w.symbol、.accentable()、.privacy_sensitive() 和 .redacted()。
