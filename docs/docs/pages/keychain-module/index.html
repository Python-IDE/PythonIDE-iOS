iOS 原生模块 / 系统服务

keychain

iOS 钥匙串安全存储。

keychain 用 iOS 钥匙串保存 token、密码、API Key 和长期凭据。它适合敏感小字符串，不适合主题、筛选条件、普通 JSON 或大文件。

#预期效果

示例会展示敏感字段写入钥匙串、读取和删除，并避免把明文打印到日志。

#什么时候用

  • 保存登录 token、第三方 API Key、私密笔记密码。
  • 退出账号、切换环境或删除连接时清理凭据。
  • 列出当前 App 保存过的 service/account 组合，帮助用户管理凭据。

#核心模型

钥匙串条目由 service 和 account 两个字符串定位。service 通常是稳定命名空间，例如 openai、github、myapp.production；account 是用户名、邮箱或环境名。写入用 set_password，读取用 get_password，删除用 delete_password。

不要把 get_password 的结果打印到用户可见日志里。需要展示时只显示“已保存”“未配置”或掩码。

#AppUI 中怎么接

AppUI 里输入 token 时，用 SecureField 收集，点击保存按钮后再调用 set_password。不要在 body() 里读取并展示明文 secret。

#与 storage 的区别

数据推荐模块
主题、筛选条件、小型 JSONstorage
API Key、token、密码keychain
大文件、图片、导出结果文件系统或对应媒体模块

#失败路径

情况应该怎么处理
读取不到凭据显示“未配置”，引导用户重新输入。
保存返回 False保留输入状态，提示保存失败。
用户退出账号调用 delete_password(service, account)。
需要保存多个账号使用稳定的 service/account 组合，不要覆盖同一个 key。

#常见错误

错误写法后果修正
把 token 保存到 storage敏感数据不安全使用 keychain.set_password。
打印 get_password() 的返回值secret 进入日志只打印是否已配置。
每次运行随机生成 service之后无法读取旧凭据使用固定命名。
