coreml

Core ML 模型加载、图片推理和模型信息读取。

coreml 用来加载 Core ML 模型、执行图片分类推理，并读取模型元数据。

#预期效果

示例会展示模型列表、模型加载、图片推理和不可用时的兜底提示。

#什么时候用

  • 运行已经打包到 App 或用户工作区里的 Core ML 图片分类模型。
  • 列出可用模型，让用户选择一个模型后再推理。
  • 在展示结果前读取模型说明、输入输出信息或作者元数据。

#核心模型

list_models() 返回模型名称字符串列表。先用 load_model(name) 得到模型句柄，再把本地图片路径传给 predict_image(model, image_path)。当前文档只覆盖推理，不覆盖模型训练或在线下载模型。

predict_image(...) 接收的是图片文件路径，不是 PIL.Image、bytes 或 Base64。图片来自相册时，先保存到可访问路径，或者使用适合图片对象的其它模块。

#快速开始

#AppUI 中怎么接

加载和推理都可能耗时，按钮回调里要显示状态。下面的示例不在页面构建时加载模型。

#图片推理配方

当用户已经有一张可访问的本地图片时，可以把模型选择、图片路径和推理结果放进同一个 AppUI 页面。示例默认使用 list_models() 的第一个模型。

预期效果：页面允许输入图片路径并手动运行推理；没有模型、路径错误或空结果都会显示为页面状态。

#公开 API

#coreml.list_models() -> list[str]

返回可用模型列表，包括 App 内置模型和用户添加的 .mlmodelc 文件。

#coreml.load_model(name) -> Model

按名称加载模型。name 可以是模型名称或可访问的模型路径。

#coreml.predict_image(model, image_path) -> list[dict]

对图片分类模型执行推理，返回 top-5 结果。

#coreml.model_info(model) -> dict

获取模型元数据，例如输入输出规格、作者、描述等。

#失败路径

情况应该怎么处理
list_models() 返回空列表显示“没有可用模型”，不要继续 load_model。
load_model(name) 抛错检查模型名称是否来自 list_models()。
图片路径不可访问让用户重新选择或保存图片到可访问位置。
推理结果为空显示空状态，不要伪造结果。

#常见错误

错误写法后果修正
models[0]["name"]list_models() 返回字符串，代码会报错使用 models[0]。
传入图片对象或 Base64predict_image 需要路径传本地图片文件路径。
在 AppUI body() 里加载模型页面刷新时重复加载放到按钮或明确任务回调里。

#使用规则

  • 当前支持已编译的 .mlmodelc 模型推理，不支持训练。
  • 大模型首次加载可能较慢，AppUI 中要给出加载状态。
  • 图片路径必须是当前脚本可访问的文件路径。
  • 不要调用以下划线开头的未公开接口，它们不是用户 API。
