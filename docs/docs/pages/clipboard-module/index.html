clipboard

读取、写入和清空系统剪贴板。

clipboard 用来读取、写入和清空系统剪贴板。它适合明确的复制、粘贴、分享前准备动作，不适合做持久化存储。

#预期效果

示例会展示读取、写入和清空系统剪贴板，并把结果反馈到界面。

#什么时候用

  • 用户点击“复制链接”“复制代码”“粘贴文本”。
  • 在导出、分享或调试前临时放入一段文本。
  • 也支持 get_text()、set_text() 兼容别名。

#核心模型

剪贴板是系统级共享状态，可能被其他 App 改写，也可能包含用户隐私内容。写入必须来自明确动作，写入后要给出可见反馈。读取为空时返回空字符串。

#最小正确示例

#AppUI 中怎么接

复制和粘贴都放在按钮回调里，界面要显示当前结果。

#失败路径

情况应该怎么处理
剪贴板为空显示“剪贴板为空”，不要覆盖当前输入。
用户要复制敏感内容明确提示风险，优先用 keychain 保存。
其他 App 改写剪贴板每次粘贴时重新读取，不缓存旧值。
只需要长期保存文本使用 storage 或文件系统。

#常见错误

错误写法后果修正
启动脚本就写入剪贴板静默覆盖用户内容只在用户点击后写入。
用剪贴板保存 token容易被其他 App 或用户误粘贴使用 keychain。
复制后没有提示用户不知道是否成功更新 State 或显示 HUD。

类型API签名说明
