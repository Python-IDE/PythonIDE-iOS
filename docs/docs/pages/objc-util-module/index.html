objc_util

Objective-C Runtime 调用与系统框架访问。

objc_util 是 Objective-C Runtime 访问层，用来访问系统类、selector、Foundation/UIKit 对象、block、ctypes 类型和少量 CoreGraphics 结构。优先使用已经封装好的模块；只有缺少高级封装时再用它。

#预期效果

示例会展示 Objective-C Runtime 类访问、Python 对象转换和主线程调用的安全边界。

#安全使用模型

  1. 先找公开封装模块；只有确实缺少能力时再使用 objc_util。
  2. 系统类名、selector 和参数类型要逐项确认，不猜测方法签名。
  3. UIKit 相关调用放在 @on_main_thread 中执行。
  4. block、回调对象和长期持有对象用 retain_global 保活。
  5. 不在示例里访问私有 API，不绕过系统权限和用户动作要求。

#失败路径

情况应该怎么处理
类名或 selector 不存在先确认系统版本和类名，再调用 ObjCClass 或方法。
UI 调用崩溃或无效用 @on_main_thread 包住相关函数。
回调对象提前释放用 retain_global 保活，流程结束后再 release_global。
普通能力实现太复杂优先使用 photos、network、location、appui 等已封装模块。

#使用规则

  • 不要调用未确认签名的私有 API。
  • UIKit 操作放到 on_main_thread。
  • 回调 block 要用 ObjCBlock 并用 retain_global 保活；不用后 release_global。
  • 传 URL 用 nsurl()，传字符串/列表/字典用 ns()。
  • objc_util 是高级能力入口，不要把普通 UI、网络、照片、定位逻辑都写成 Runtime 调用。
