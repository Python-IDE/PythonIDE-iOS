#先选模块

目标首选模块注意
查询或申请权限permission只在用户点击后申请，不在页面构建时弹窗。
选择、拍照、保存图片photos用户取消时会返回空值，要先判断。
保存 token、密码和密钥keychain不要把 secret 写到 storage 或日志里。
保存普通设置storage适合主题、开关、筛选条件等非敏感数据。
弹窗、HUD、输入提示console / dialogs短反馈用 HUD，结构化输入用 dialogs。
定位、运动、健康、蓝牙、NFC对应能力模块先展示用途，再请求权限或开始扫描。
网络、下载、WebSocketnetwork / websocket复杂请求不要放在 AppUI body() 中。
系统自动化入口shortcuts / live_activity必须由明确用户动作触发。

#完整工作流示例

下面的 AppUI 示例把“选择照片、复制结果、保存配置状态”放进按钮回调里。页面构建只读取状态，不触发系统面板。

预期效果：页面显示照片和配置状态；点击照片按钮后系统选择器出现，取消会保留页面，选择成功会把图片大小复制到剪贴板；点击保存按钮只展示保存状态，不暴露 token 明文。

#平台与权限矩阵

能力真机要求权限用户动作推荐入口
相册选择、拍照、保存推荐真机photos / camera是photos
位置、指南针、运动真机location / motion是location、motion
HealthKit真机health是health
BLE 扫描和连接真机bluetooth是bluetooth
NFC 读取和写入真机且设备支持 NFCnfc是nfc
本地通知真机/模拟器均可测试部分流程notifications是notification
Vision OCR 和检测取决于系统能力无通常是vision、vision_helper
Core ML 推理取决于模型和设备无否coreml
WebSocket网络可用即可无否websocket
Keychain、Storage、ClipboardApp 环境无Clipboard 推荐用户动作keychain、storage、clipboard

需要系统面板、硬件扫描或读取隐私数据的能力，都应该由按钮、菜单、刷新或系统动作触发；不要在 AppUI body() 或模块导入时自动执行。

#失败路径

情况应该怎么处理
权限被拒绝显示当前状态和授权说明，不要循环弹出系统授权。
用户取消系统面板处理 None、False、空列表或错误字典，保持页面可继续操作。
设备或系统版本不支持显示不可用状态，并提供替代流程或禁用对应按钮。
返回数据为空把空状态写回界面，不把空结果当作异常崩溃。

#发布前检查

检查项合格标准
用户动作权限、照片、蓝牙、NFC、通知和快捷指令都由按钮或菜单触发。
空结果对 None、False、空列表、空字符串和错误字典都有页面反馈。
权限被拒绝后显示当前状态和设置入口，不循环申请。
隐私token、联系人、健康、位置和照片内容不写入公开日志。
状态AppUI body() 不执行系统调用，只读取 State。
