scene 概览

2D 场景、节点、动画、物理与绘制。

scene 是 Pythonista 风格的 2D 场景运行时，适合小游戏、粒子、Sprite、触摸交互、持续绘制和物理效果。普通表单、设置页和业务工具优先用 appui。

#预期效果

运行示例后会打开 2D 场景，显示可点击坐标、循环动画精灵和 Classic 绘制图形。

#先选写法

需求首选原因
小游戏、Sprite、粒子、碰撞、触摸循环scene有持续帧循环、节点树、Action 和 Physics。
普通工具页、设置页、列表、图表和媒体控制appui原生控件和状态管理更稳定。
Pythonista 风格命令式窗口ui适合迁移命令式脚本和简单自定义控件。
主屏、锁屏或 StandBy 展示widget系统小组件不是实时场景，必须用 WidgetKit 模型。
只需要教材式 turtle 绘图turtleAPI 更接近教学脚本，不需要节点和物理。

#写 scene 的心智模型

  1. 入口：定义 Scene 子类，脚本末尾调用一次 scene.run(MyScene())。
  2. 初始化：setup() 里创建节点、纹理、物理体和稳定引用。
  3. 持续逻辑：update() 里只做轻量状态推进，不做网络请求或大文件读写。
  4. 自绘：draw() 适合 Classic 绘图；节点动画优先用 Action。
  5. 触摸：touch_began、touch_moved、touch_ended 只更新场景状态。
  6. 资源：图片、音效和控制器状态要缓存，避免每帧重复加载。

#失败路径

情况应该怎么处理
场景不显示确认最后调用 scene.run(SceneSubclass())，并且 setup() 没有抛错。
触摸无反应使用 touch_began / touch_moved / touch_ended，不要把事件逻辑写进普通函数后忘记绑定。
帧率下降不要在 update() 或 draw() 中做网络请求、文件大读写或反复创建大量节点。
普通表单很难做表单、设置、列表和导航页改用 appui。

#发布前检查

检查项合格标准
入口脚本末尾只调用一次 scene.run(...)。
生命周期节点在 setup() 创建，持续变化在 update() 或 Action 中完成。
帧循环每帧不创建大量节点、不重复加载图片、不做阻塞 I/O。
触摸触摸方法使用 touch.location，并能处理多次点击或拖动。
边界普通表单、设置和长列表没有写进 scene。

#使用规则

  • setup() 创建节点和资源，update() 放持续逻辑，draw() 放经典绘制，触摸逻辑放 touch_began / touch_moved / touch_ended。
  • 不要把普通数据列表或设置页写成 scene；那类界面用 AppUI 的 List / Form。
  • 节点动画优先用 Action，不要在 update() 里重复创建节点。
  • 物理和触摸对象要用稳定引用，避免每帧重新分配大量对象。
