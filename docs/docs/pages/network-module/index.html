network

原生 HTTP 请求、连接状态和下载。

network 提供原生 HTTP 请求、网络状态判断和文件下载。它适合脚本内直接访问 REST API，也适合 AppUI MiniApp 在点击、刷新、搜索时拉取数据。

#预期效果

示例会展示联网判断、GET/POST/下载，以及 AppUI 请求中的加载、成功和失败状态。

#什么时候用

  • 读取 JSON API、提交表单、上传小型请求体。
  • 判断当前是否联网、是否低数据模式、是否蜂窝网络。
  • 下载文件到脚本可访问的目标路径。

#API 参考

API返回用途
is_connected()bool当前是否有可用网络。
connection_type()strwifi、cellular、ethernet、none、other。
is_expensive()bool是否蜂窝网络或热点等高成本网络。
is_constrained()bool是否开启低数据模式。
request(method, url, headers=None, body=None, json=None, timeout=30)Response通用 HTTP 请求。
get/post/put/delete/patch(url, **kwargs)Response常用方法快捷入口，参数同 request。
download(url, dest_path, timeout=120)bool下载成功返回 True，失败返回 False。
Response.okboolstatus 在 200 到 299 之间时为真。
Response.statusintHTTP 状态码。
Response.textstr响应文本。
Response.json()Any按 JSON 解析响应；非 JSON 会抛解析错误。
Response.headersdict[str, str]响应头。

#失败路径

情况应该怎么处理
is_connected() 为 False显示离线状态，不自动重试大请求。
response.ok 为 False展示 response.status 和简短错误内容。
JSON 解析失败捕获异常，改为显示 response.text 摘要。
download(...) 返回 False保留旧文件或提示重新下载。

#使用规则

  • 请求前可以用 is_connected() 给用户更清楚的错误提示，但最终仍要处理请求失败。
  • 所有 UI 按钮触发的请求都要显示加载、成功和失败状态。
  • response.json() 只在你确认接口返回 JSON 时调用，并捕获异常。
  • 蜂窝网络或低数据模式下，避免自动下载大文件；先询问用户或改成手动触发。
