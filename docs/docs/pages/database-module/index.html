database

MiniApp 作用域内 SQLite 数据库和 JSON collection。

database 是 MiniApp 作用域内的 SQLite 持久化模块，适合保存可查询的历史记录、收藏、播放进度、离线缓存、源列表和业务表。小型设置继续用 storage，敏感 token 或密码用 keychain。

#预期效果

示例会展示 SQLite / collection 数据的创建、写入、查询和失败处理。

#什么时候用

目标首选 API
简单 JSON 记录、收藏、历史、缓存database.collection(name)
需要索引、排序、JOIN、迁移database.open(name) + SQL
少量开关、主题、最后一次选择storage
密码、token、密钥keychain

数据库文件会放在当前 MiniApp 的数据目录下。database.open("media") 会自动变成 media.db，并拒绝绝对路径和 ../，避免脚本越权写到其他位置。

#Collection API

Collection 是大多数 MiniApp 的首选。它用 SQLite 存 JSON 文档，避免用户为了普通列表和缓存手写建表。

API说明
collection(name)返回默认数据库里的 Collection。
Collection.upsert(key, value)插入或更新 JSON 值。
Collection.get(key, default=None)读取 JSON 值，不存在时返回默认值。
Collection.delete(key)删除一条记录。
Collection.clear()清空当前 collection。
Collection.count()当前 collection 记录数。
Collection.items(order_by="updated_at desc", limit=None, offset=0)返回包含 key、value、created_at、updated_at 的记录。
Collection.list(order_by="updated_at desc", limit=None, offset=0)只返回 value 列表。
Collection.migrate_from_storage(storage_key, key_field="id", remove=False)把旧 storage JSON 迁移成 collection 记录。

#SQL API

需要明确表结构、索引或事务时使用 Database。

API说明
open(name="app.db", **kwargs) / connect(...)打开 MiniApp 作用域内的 SQLite 数据库。
default_database()返回默认数据库实例。
database_path(name="app.db")返回数据库绝对路径，用于调试和导出。
Database.execute(sql, params=None)执行一条 SQL，返回 affected row count。
Database.executemany(sql, seq_of_params)批量执行。
Database.executescript(script)执行建表或迁移脚本。
Database.query(sql, params=None)返回 list[dict]。
Database.query_one(sql, params=None, default=None)返回单行 dict 或默认值。
Database.scalar(sql, params=None, default=None)返回第一列标量。
Database.transaction()原子事务，异常时回滚。
Database.collection(name)在这个数据库里创建 collection。
Database.user_version读写 SQLite PRAGMA user_version，适合记录 schema 版本。
Database.close()关闭连接。

#常见错误

错误正确做法
把大量列表、播放历史、搜索缓存塞进 storage用 database.collection(...)。
把 token 或密码放进 SQLite用 keychain。
传入 ../data.db 或绝对路径只传文件名，例如 database.open("media")。
长时间不用还保持连接页面或任务结束时调用 db.close()。
手写 SQL 拼接用户输入使用 ? 参数绑定。
