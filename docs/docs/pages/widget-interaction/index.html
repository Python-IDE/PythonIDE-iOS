widget / 小组件

状态和交互

AppIntent 按钮、Toggle、链接和 widget.state。

桌面 Widget 不执行任意 Python 回调。按钮和开关只能触发受控 AppIntent：修改 widget.state、刷新小组件，或打开链接。

#计数器

预期效果：点击 + 或 - 修改计数，Toggle 保存完成状态；所有交互文字都保持在按钮或行内。

#状态类型

状态适合
widget.state.int次数、步数、分数
widget.state.float进度、价格、比例
widget.state.bool完成、开启、收藏
widget.state.str当前模式、选中项
widget.state.list勾选列表、Bingo、习惯打卡

#常用动作

  • count.increment()、count.decrement()、count.set(value)。
  • done.toggle()。
  • items.toggle_item(item)。
  • w.button(..., action=...)。
  • w.toggle(..., state=...)。

#链接

预期效果：显示一个链接入口，点击后打开 URL；链接标题短且不会和其他交互区域重叠。

如果一个容器里已经有按钮或开关，不要再给整个容器加链接。交互区域应该清楚：整块链接、局部链接、按钮、开关只能按层级明确共存。

#失败路径

  • 按钮点击没反应：action 必须来自 widget.state，例如 count.increment()、done.toggle() 或 items.toggle_item(i)。
  • Toggle 显示但不保存：优先写 w.toggle("完成", state=done)，不要把静态布尔值和手写 action 拼在一起。
  • 链接和按钮冲突：整块链接、局部链接、按钮、开关要分清层级；同一块区域不要同时承担多个动作。
  • 点击后桌面仍旧：AppIntent 已执行但 WidgetKit 可能缓存显示，重新运行并发布；必要时删除桌面小组件后重新添加。

#API 参考

交互完整签名见 API 参考：重点查 widget.state、w.button、w.toggle、w.link、.pressed() 和 .intent()。
