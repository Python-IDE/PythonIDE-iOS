原生能力入口

MiniApp 常用原生能力入口，不是完整模块总表。

这里按 MiniApp 场景整理常用原生能力入口。它不是完整模块总表；完整签名仍以各模块文档为准。

#预期效果

示例会展示 AppUI 页面如何调用原生模块，并把成功、取消和失败状态写回界面。

#使用原则

  • 原生能力由用户动作触发：按钮、刷新、选择器、页面明确操作。
  • body() 只负责描述界面，不要在里面请求权限、发通知、启动传感器或做网络请求。
  • 权限、失败、空结果都要写回 State，让用户看见当前状态。
  • 设置类数据用 storage，可查询记录和缓存用 database，敏感数据用 keychain，大文件用文件系统。

#场景配方

目标推荐组合说明
选择照片并保存设置appui + photos + storage按钮触发 picker，结果路径或元数据写入 State。
下载视频并保存到相册network + photosnetwork.download(...) 写入本地文件，再用 photos.save_video(path) 保存。
地图页展示当前位置permission + location + appui.MapView先查权限，再请求定位，地图只展示状态结果。
健康数据面板permission + health + appui.Chart权限被拒绝时显示空状态，不要让图表空白。
提醒或定时任务notification + storage保存用户设置，再按稳定 identifier 创建或取消提醒。
播放历史或收藏列表database + appui.List用 collection 保存多条记录、进度和更新时间，不要把大列表塞进 storage。
安全配置页biometric + keychain + appui.Form密钥进 keychain，解锁动作走生物认证。
实时传感器面板motion + haptics + ReactiveState高频数据不要每帧重建整棵 UI。
远端列表network + appui.List + .refreshable网络请求放在刷新/按钮回调，结果写回 State。

#AppUI 中调用原生能力

原生能力应该由按钮、刷新、生命周期等明确交互触发，并把结果写回 State。不要在 body() 里直接请求权限、发通知或做网络请求。

#原生能力组合示例

这个示例把设备信息、通知、触觉和本地持久化组合到一个 AppUI 页面里：按钮触发系统能力，结果写回 State，快照用 storage.set_json(SNAPSHOT_KEY 保存。

#使用规则

  • 权限类能力先查授权状态，再请求权限。
  • 持久化设置用 storage，查询型记录、历史、收藏和缓存用 database，密码、token、密钥用 keychain。
  • 触觉、通知、声音属于反馈，不要在循环里连续触发。
  • 相册、定位、通知这类能力必须给用户明确动作入口。
  • 写代码前先打开对应模块文档，按公开 Python API 调用，不要凭通用 iOS 记忆猜接口名。

#失败路径

情况UI 应该怎么处理
权限未授权显示说明和授权按钮，不要直接展示空白页面。
用户取消选择保持原状态，并给出简短状态文本。
设备不支持用 ContentUnavailableView 或 Section footer 说明原因。
网络失败保留旧数据，显示错误信息和重试动作。
数据为空显示空状态，不要把空列表误认为加载中。
