notification

本地通知权限、定时通知和角标管理。

notification 用来请求本地通知权限、调度延迟通知或指定日期通知、查看待发送通知、取消通知和设置角标。它只负责本地通知，不负责远程推送。

#预期效果

示例会展示通知授权、定时本地通知、取消通知和角标更新。

#什么时候用

  • 设置提醒、倒计时、番茄钟、稍后处理提示。
  • 让用户查看或取消当前脚本创建的待发送通知。
  • 更新 App 角标数量。

#核心模型

先调用 request_permission()，再调度通知。schedule() 使用相对延迟秒数；schedule_at_date() 使用年、月、日、小时、分钟参数，不接受时间戳。identifier 应该稳定，这样重复调度、更新和取消都能定位同一条通知。

如果只想统一查询权限状态，也可以用 permission 的 permission.status("notifications")。

#指定日期示例

schedule_at_date 的日期参数是拆开的，不是 Unix 时间戳。

#AppUI 中怎么接

调度和取消都放在按钮回调里，界面显示权限、待发送数量和最近操作。

#失败路径

情况应该怎么处理
用户拒绝通知权限显示未授权状态，不继续调度。
schedule 返回错误字典展示错误信息，保留按钮可重试。
用户修改提醒时间用同一个 identifier 重新调度或先 remove_pending。
不再需要提醒调用 remove_pending 或 remove_all_pending。

#常见错误

错误写法后果修正
不请求权限直接调度通知可能不会出现先 request_permission()。
schedule_at_date(..., timestamp)参数不匹配传 year, month, day, hour, minute。
每次生成随机 identifier无法取消或更新旧通知使用稳定 ID。
把远程推送写进本模块示例能力边界错误本模块只做本地通知。

#API 参考

类型API签名说明
functionrequest_permissionrequest_permission() -> Any请求本地通知权限，返回结果字典。
functionauthorization_statusauthorization_status() -> Any返回当前通知授权状态字符串。
functionscheduleschedule(identifier, title, body, delay=5.0, sound=True, badge=0) -> Any在 delay 秒后调度本地通知。
functionschedule_at_dateschedule_at_date(identifier, title, body, year, month, day, *, hour=0, minute=0, sound=True, badge=0, repeats=False) -> Any在指定年月日时分调度本地通知。
functionremove_pendingremove_pending(identifier) -> None删除指定 ID 的待发送通知。
functionremove_all_pendingremove_all_pending() -> None删除所有待发送通知。
functionpending_countpending_count() -> Any返回待发送通知数量。
functionpending_identifierspending_identifiers() -> Any返回待发送通知 ID 列表。
functionset_badgeset_badge(number) -> None设置 App 角标数量。
