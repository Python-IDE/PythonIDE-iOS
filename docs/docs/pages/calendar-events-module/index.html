calendar_events

日历事件、提醒事项和 EventKit 权限。

calendar_events 用 EventKit 读取和写入日历事件、提醒事项，并查询对应权限。事件和提醒事项是两个独立权限：写事件前请求 request_access()，写提醒前请求 request_reminder_access()。

#预期效果

示例会展示日历/提醒授权、事件创建、查询、删除和未授权处理。

#什么时候用

  • 创建会议、专注时间块、行程事件，并写入用户选择的日历。
  • 查询一段时间内的事件，做日程摘要或仪表盘。
  • 创建、完成或删除提醒事项。

#核心模型

时间参数使用 Unix timestamp。get_events(start, end, calendar_id=None) 必须传起止时间；create_event(title, start, end, ...) 的 end 应晚于 start。calendar_id 可来自 get_calendars() 返回的日历列表，不传时使用系统默认写入目标。

创建类 API 通常返回字典，删除和完成 API 返回布尔值。代码应处理未授权、能力不可用、空列表和用户取消。

#失败路径

情况应该怎么处理
日历或提醒事项未授权显示当前状态，不继续写入。
get_events 返回空列表显示“该时间段没有事件”。
end <= start阻止创建，提示检查时间。
删除或完成返回 False保留原列表状态，提示操作失败。

#常见错误

错误写法后果修正
写提醒前只请求日历权限提醒事项仍无权限使用 request_reminder_access()。
把日期字符串传给 create_event时间解析失败或写错时间传 Unix timestamp。
不保存事件 id后续无法删除指定事件保存返回结果里的 id 字段。
